# DESIGN.md

ステータス: v0 Draft  
更新日: 2026-04-20  
用途: 次セッションで `DESIGN.md v1` を確定するための初期叩き台

## 1. このファイルの役割

この `DESIGN.md` は、実装仕様書ではない。  
竹村英子ホームページ刷新において、画面実装より前に

- 何をどう見せるか
- どの温度感で伝えるか
- BtoC / BtoB をどう同居させるか

を固定するためのデザイン制御ドキュメントとして扱う。

## 2. この案件で確定している戦略前提

- サイトは `竹村英子` を主語にした 1 サイト構成
- BtoC を主軸、BtoB を第2の柱にする
- 個人向け主力商品は `じぶん整え習慣サロン`
- `カラーココロジー研究所` は屋号として残す
- `ごきげん体質` は今後使わない
- 個人向け主ターゲットは `40代以上女性`
- 法人向け主ターゲットは `企業人事`
- 個人向け主入口は `無料診断`
- 診断後はメールナーチャリングを通じて `無料相談` やサロン参加へ接続する
- サロン申込の外部LPは、いきなり見せず、説明や関係構築の後で案内する

## 3. サイトが解決すべきこと

現行サイトでは、

- 何をしている人なのか
- 個人向けなのか法人向けなのか
- いま一番案内したいサービスが何なのか

が初見で掴みにくい。

新サイトでは、次を達成する必要がある。

1. 竹村英子さんの専門性を短時間で理解できる
2. 個人向け / 法人向けの導線が迷わず選べる
3. 個人向けでは `じぶん整え習慣サロン` へ自然に関心が向く
4. 法人向けでは `相談できる専門家` として信頼が立つ
5. 実績が `何でも屋` ではなく `整える専門家` として見える

## 4. 優先順位

### 優先度 A

- トップ
- 個人向けトップ
- `じぶん整え習慣とは`
- `じぶん整え習慣サロン`
- `無料診断`
- 法人向けトップ

### 優先度 B

- 実績・お客様の声
- 竹村英子について
- 研修メニュー
- 解決できる課題

### 優先度 C

- コラム / お知らせ
- FAQ
- 更新追加される周辺ページ

## 5. 表現の主語

- ブランドの主語は `竹村英子`
- 屋号の `カラーココロジー研究所` は信頼資産として扱う
- `コミュニケア®` は方法論のひとつとして扱う
- サイト全体を、診断メソッドや講座名の集合体にはしない

## 6. 絶対に外したくない印象

- 整っている
- 信頼できる
- 押しつけがましくない
- 女性向けでも軽すぎない
- 専門家だが冷たくない
- 健康や心の話でもスピリチュアルに寄りすぎない

## 7. 避けるべき見え方

- 量産型の自己啓発サイト
- ふわっとしたスピリチュアル表現
- かわいさ優先で信頼が落ちる見え方
- 法人向けに寄せすぎて個人向けの親しみが消える見え方
- サービスを並べすぎて主力が見えない見え方
- 実装都合で均質化されたテンプレート感

## 8. デザイン原則

1. まず `整える人` として見えること
2. 余白と情報整理で安心感を出すこと
3. BtoC は親密さ、BtoB は信頼感を出し分けること
4. ただし別サイトのように分断しないこと
5. CTA は強引に押さず、理解の流れの中で配置すること
6. 外部LPへの遷移は、説明後の納得導線として扱うこと

## 9. 次セッションで決めるべき項目

### 9-1. トンマナ方向

次の 3 方向から、1本に寄せて決める。

1. Calm Editorial
   - 整理された誌面感
   - 知性と落ち着きが先に立つ
   - 企業向けとの接続がしやすい
2. Warm Holistic
   - やわらかく寄り添う印象
   - 個人向けとの相性が高い
   - ただし軽く見せすぎない制御が必要
3. Refined Natural
   - 自然感と上質感の両立
   - 健康テーマと相性がよい
   - 法人向けの硬さ不足に注意

### 9-2. 写真 / ビジュアル方向

- 竹村英子さん本人写真の主役度
- イラストより写真を主軸にするか
- 色・食・体の要素をどの程度ビジュアル化するか
- 法人向けで使う写真のトーン

### 9-3. 情報密度

- トップは軽く見せるか、実績までしっかり見せるか
- 個人向けページは共感先行か、説明先行か
- 法人向けページは課題訴求先行か、研修内容先行か

### 9-4. CTA の圧力

- `無料診断` をどこまで主役に出すか
- `無料相談` は補助導線としてどこに置くか
- 外部LPへの遷移ボタンをどう見せるか

## 10. 実装前に守ること

- まだ CSS 設計を確定しない
- まだコンポーネント一覧から入らない
- まだ CMS の詳細設計に入らない
- まだ画面数を増やしすぎない
- まずトップ / 個人向け / 法人向けの視覚原則を固める
